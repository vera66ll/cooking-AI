# CloudBase 云开发配置指南

## 📋 配置步骤

### 1️⃣ 开通云开发服务

1. 打开**微信开发者工具**
2. 点击顶部菜单栏的 **云开发** 按钮
3. 首次使用需要开通云开发服务
4. 选择**基础版**（免费额度足够使用）
5. 等待环境创建完成

### 2️⃣ 获取环境ID

1. 在云开发控制台中，找到 **环境设置**
2. 复制您的**环境ID**（格式类似：cloud1-xxx）

### 3️⃣ 修改配置文件

打开 `miniprogram/app.ts` 文件，找到第10行：

```typescript
env: 'your-env-id', // TODO: 替换为您的云环境ID
```

将 `'your-env-id'` 替换为您实际的环境ID，例如：

```typescript
env: 'cloud1-abc123', 
```

### 4️⃣ 上传云函数

在微信开发者工具中：

1. 右键点击 `cloudfunctions` 文件夹
2. 选择 **当前环境** -> 选择您的云环境
3. 依次右键每个云函数文件夹：
   - `userLogin`
   - `getUserInfo`
   - `updateUserInfo`
   - `initDatabase`
4. 选择 **上传并部署：云端安装依赖**

### 5️⃣ 初始化数据库

有两种方式初始化数据库：

#### 方式一：通过云函数（推荐）

在微信开发者工具的**控制台**中执行：

```javascript
wx.cloud.callFunction({
  name: 'initDatabase'
}).then(res => {
  console.log('数据库初始化结果:', res);
});
```

#### 方式二：手动创建

在云开发控制台的**数据库**页面，手动创建以下集合：
- users
- dishes
- favorites
- menus
- comments

### 6️⃣ 配置数据库权限

在云开发控制台的**数据库**页面：

1. 点击每个集合
2. 点击 **权限设置**
3. 选择 **自定义安全规则**
4. 参考 `cloudfunctions/database-rules.json` 中的配置

或者使用默认权限：**所有用户可读，仅创建者可读写**

## ✅ 验证配置

### 测试云函数

在小程序任意页面的 `onLoad` 中添加测试代码：

```javascript
wx.cloud.callFunction({
  name: 'userLogin'
}).then(res => {
  console.log('云函数调用成功:', res);
}).catch(err => {
  console.error('云函数调用失败:', err);
});
```

如果看到成功日志，说明配置正确！

## 🔧 常见问题

### Q1: 云函数调用失败？
- 检查环境ID是否正确
- 确认云函数已上传
- 查看云函数日志排查错误

### Q2: 数据库操作失败？
- 检查集合是否已创建
- 确认权限设置正确
- 查看控制台错误信息

### Q3: 如何查看云函数日志？
- 云开发控制台 -> 云函数 -> 选择函数 -> 日志
